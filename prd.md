📄 PRD – Website Prototype Feature List

(Product Requirements Document – Research Version)

Product Name (Example)

CalmUX Lab – Experimental Interaction Platform

🎯 Goal

To experimentally evaluate how calm technology design features affect emotional response, usability, and reflection using Norman’s three processing levels.

👥 Target Users

Students

Knowledge workers

General digital users (18–35)

🔧 Core Website Modules (All Conditions Share Same Tasks)
1. User Dashboard

Daily system status

Progress tracking

Notifications

Data visualization

2. Task Area

Monitoring changes

Responding to prompts

Completing timed activities

3. Reflection Journal

Post-task thoughts

Memory recall prompts

🎨 Feature Mapping by Experimental Condition
🟥 Control UI (Traditional Attention-Based)

Pop-up notifications

Bright alerts

Explicit text updates

Frequent reminders

🌿 Visceral Calm Features
Emotional + Sensory

Soft animated gradients

Ambient color shifts for status

Micro-animations instead of alerts

Soundless motion cues

Example Features:

Mood-color background

Pulsing indicators

Calm transitions

⚙️ Behavioral Calm Features
Usability + Flow

Peripheral progress rings

Subtle edge indicators

Non-blocking notifications

Glanceable information

Example Features:

Sidebar ambient bars

Background progress waves

Hover-based details

🌊 Reflective Calm Features
Meaning + Attachment

Nature metaphors (water, light, growth)

Personal data storytelling

Memory timeline

Reflection prompts

Example Features:

Flowing time visualization

“Your calm journey” page

Weekly meaning summary

📊 Research Instrument Integration
Built-in Metrics

Task timers

Error logging

Interaction heatmaps

Survey popups

📐 Success Metrics (KPIs)
Level	Metric
Visceral	Stress ↓, positive affect ↑
Behavioral	Speed ↑, errors ↓
Reflective	Recall ↑, attachment ↑

Anonymous IDs

Opt-out anytime
